# Подсистема времени прошивки Nixie Clock

## Назначение
Документ описывает реализацию ядра времени: выбор источника, чтение UTC, синхронизацию и преобразование.

## Компоненты
- `include/time_utils.h`
- `src/time_utils.cpp`
- `include/hardware.h`
- `src/timezone_manager.cpp`
- `include/timezone_manager.h`
- `src/config.cpp`

## Основные функции

### checkTimeSource()
Отвечает за:
- загрузку I2C: `Wire.begin(I2C_SDA, I2C_SCL)`
- проверку доступности DS3231
- создание/инициализацию `RTC_DS3231`
- проверку корректности времени DS3231
- переключение `currentTimeSource`
- запуск прерываний `setupInterrupts()`

Особенности:
- проверка выполняется при каждом вызове `getCurrentUTCTime()`
- при отсутствии DS3231 используется внутренняя RTC
- при обнаружении нового DS3231 происходит синхронизация времени
- поддерживается детекция флага OSF и очистка ошибки питания

### getCurrentUTCTime()
- основной интерфейс для всех модулей
- сначала вызывает `checkTimeSource()`
- если активен DS3231 — возвращает его время
- иначе возвращает системное время ESP32

### setTimeToAllSources(time_t utcTime)
- устанавливает время в системный RTC ESP32
- если DS3231 доступен — обновляет внешний RTC
- используется при ручной установке и при синхронизации

### setDefaultTimeToAllSources()
- когда время внешнего RTC невалидно
- устанавливает базовое время по умолчанию
- предотвращает использование некорректной даты

### convertDateTimeToTimeT() / convertTimeTToDateTime()
- преобразуют `DateTime` (RTClib) в `time_t` и обратно
- `convertDateTimeToTimeT()` формирует `struct tm` в UTC
- `convertTimeTToDateTime()` строит объект `DateTime` из UTC

### syncTimeAsync(bool force, uint8_t preferredNtpIndex)
- запускает асинхронную синхронизацию времени через FreeRTOS
- создаёт задачу на ядре 0 через `xTaskCreatePinnedToCore`
- выполняется только если WiFi настроен и синхронизация разрешена
- если `force == true`, игнорирует флаг `auto_sync_enabled`

Проверки перед запуском:
- если `timeClient` не инициализирован — синхронизация не стартует
- если WiFi не настроен — синхронизация невозможна
- если авто-синхронизация отключена и `force == false` — задача не создаётся

### printTime()
- выводит UTC и локальное время
- использует `utcToLocal()` из `timezone_manager`
- печатает текущую зону, смещение и статус DST

## Источники времени

### DS3231
- приоритетный источник
- работает по I2C
- проверяется по адресу `0x68`
- время валидно, если год в пределах 2025–2100 и корректная дата

### System RTC
- резервный источник
- используется при отсутствии DS3231
- всегда доступен, но может потерять точность после перезагрузки

### NTP
- используется как источник синхронизации через WiFi
- не является мгновенным основным источником времени
- применяется для коррекции System RTC и DS3231 при подключении

## Взаимодействие с часовыми поясами

- `getCurrentUTCTime()` возвращает UTC
- для пользовательских отображений применяется `utcToLocal()` из `timezone_manager`
- при ручной установке локального времени используются функции `setManualLocalTime()` и `setManualLocalDate()` для конверсии в UTC

## Особенности реализации

- `checkTimeSource()` содержит оптимизации, чтобы не дёргать I2C слишком часто
- `getCurrentUTCTime()` защищён от рекурсии
- время DS3231 не синхронизируется в `getCurrentUTCTime()` при каждой итерации, только при смене источника и в `checkTimeSource()`
- асинхронная синхронизация запускается отдельно и не блокирует основной цикл

## Что важно задокументировать агенту

- последовательность `checkTimeSource()` → `getCurrentUTCTime()`
- разделение UTC/локального времени
- использование DS3231 как приоритета и System RTC как fallback
- отличие между `syncTimeAsync()` и обычным опросом RTC
- роль `setTimeToAllSources()` при ручной установке времени
