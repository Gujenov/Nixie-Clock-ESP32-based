# Система будильников прошивки Nixie Clock

## Назначение
Документ описывает структуру, логику и функции подсистемы будильников.

## Компоненты
- `include/alarm_handler.h`
- `src/alarm_handler.cpp`
- `include/config.h` (`AlarmSettings`)
- `src/time_utils.cpp` / `include/time_utils.h`
- `src/platform_profile.cpp`
- `src/audio_task.cpp`

## Структура данных будильников

Тип `AlarmSettings` в `include/config.h` содержит:
- `hour` — час (0..23)
- `minute` — минута (0..59)
- `enabled` — включен ли будильник
- `melody` — номер мелодии
- `days_mask` — маска дней недели для будильника 2
- `once` — одноразовый режим для будильника 1

В конфигурации хранятся два будильника:
- `config.alarm1`
- `config.alarm2`

## Основные функции

### setAlarm(alarmNum, timeStr)
- проверяет формат `HH:MM`
- сохраняет часы и минуты
- включает будильник
- пишет данные в `saveConfig()`

### setAlarmMelody(alarmNum, melody)
- обновляет номер мелодии
- сохраняет конфигурацию

### setAlarmOnceMode(alarmNum, once)
- работает только для `alarmNum == 1`
- устанавливает флаг `once`
- сохраняет конфигурацию

### setAlarmDaysMask(alarmNum, daysMask)
- работает только для `alarmNum == 2`
- сохраняет маску дней недели
- позволяет построить расписание для рабочих/выходных

### disableAlarm(alarmNum) / enableAlarm(alarmNum)
- включают или отключают конкретный будильник

### getMinutesToNextAlarm()
- вычисляет ближайшее время до следующего срабатывания
- использует локальное время
- полезно для состояния дисплея или диагностики

## Проверка срабатывания

### checkAlarms()
- вызывается регулярно из основного цикла
- получает текущее UTC через `getCurrentUTCTime()`
- конвертирует в локальное время `utcToLocal()`
- вызывает `checkAlarmsAtTick()`

### checkAlarmsAtTick(now_utc, local_timeinfo)
- срабатывает только при `local_timeinfo.tm_sec == 0`
- проверяет оба будильника
- для будильника 2 учитывает маску дней недели
- при совпадении:
  - выводит информацию в Serial
  - если настроено, запускает звуковую задачу
  - для `alarm1.once` выключает будильник после срабатывания

### Привязка к локальному времени
Будильники рассчитываются на основе локального времени, а не UTC.
Это важно, потому что для пользователя важен часовой пояс и DST.

## Логика будильников

- Будильник 1:
  - ежедневный или одноразовый
  - может иметь собственный трек мелодии
- Будильник 2:
  - ежедневный с маской дней недели
  - каждая битовая позиция соответствует дню недели

## Аудио-проигрывание

Срабатывание вызывает:
- `audioTaskStart()`
- `audioPlayAlarmMelody(melody)`

Это выполняется только если платформа поддерживает звук.

## Что важно задокументировать агенту

- модель `AlarmSettings`
- раздельные режимы будильников 1 и 2
- момент проверки: `tm_sec == 0`
- связь с локальным временем и DST
- вариант "одноразовый" для `alarm1`
